# Writing — 文案撰寫

## 觸發條件

- 撰寫部落格文章、Email 行銷文案
- 製作 Landing Page 或廣告文案
- 社群媒體貼文（LinkedIn / Instagram / Twitter）
- 需要 SEO 優化的內容
- 品牌聲音分析與風格統一

## 文案說服公式

### AIDA（認知型內容）
```
Attention  — 開頭抓住注意力（問題、數據、故事）
Interest   — 建立興趣（為什麼重要）
Desire     — 激發慾望（好處、結果）
Action     — 明確 CTA（下一步要做什麼）
```

### PAS（痛點型文案）
```
Problem — 點出讀者的痛點
Agitate — 放大問題的嚴重性
Solution — 提出你的解決方案
```

### FAB（功能轉利益）
```
Feature  — 產品功能是什麼
Advantage — 這個功能的優勢
Benefit  — 對讀者的實際好處
```

## 各類型文案範本

### 部落格文章結構
```
標題：[數字] + [具體好處] + [目標受眾]（例：5 個行銷人必知的 AI 工具）

開場（150 字）：點出痛點 / 提出問題
段落 1-N：小標題 + 核心論點 + 佐證
結尾 CTA：讓讀者做一件事（留言 / 訂閱 / 分享）
SEO：H1/H2 含關鍵字、內文密度 1-2%、meta description 120-155 字
```

### Email 行銷結構
```
主旨行：[利益] + [緊迫性] — 40 字以內，避免垃圾字（免費/限時/緊急）
預覽文字：補充主旨，增加開信率
正文：
  - 開場（個人化，點名或場景）
  - 核心訊息（1 個重點）
  - 社會證明（客戶故事 / 數據）
  - CTA 按鈕（動詞 + 利益，例：「免費試用 14 天」）
簽名：真實姓名 + 職稱 + 退訂連結
```

### Landing Page 結構
```
Hero Section：標題（value prop）+ 副標 + CTA + 視覺
社會證明：Logo 牆 / 客戶引言
功能區塊：FAB 模式（3-5 個核心功能）
比較表：vs 競品
常見問題：消除購買障礙
最終 CTA：重複首頁 CTA
```

### 社群貼文（LinkedIn）
```
第一行（Hook）：讓人想點「看更多」（數字、問句、反直覺觀點）
正文：段落短（1-3 行），每段空一行
故事 / 洞察 / 教學
結尾：問一個問題 → 引發留言
```

## 語調分析

收到既有文案時，先分析：
1. **正式度**：學術型 / 商務型 / 對話型 / 休閒型
2. **情緒調性**：激勵 / 中立 / 親切 / 權威
3. **句型偏好**：長句詳述型 vs 短句衝擊型
4. **常用詞彙**：專業術語 vs 口語

然後在輸出中匹配相同語調。

## SEO 快速清單
- [ ] 標題含主關鍵字（H1 只用一次）
- [ ] 內文關鍵字密度 1-2%（過高會被懲罰）
- [ ] Meta Description 120-155 字
- [ ] 每 300 字加一個 H2/H3
- [ ] 圖片 alt text 含關鍵字
- [ ] 內部連結 2-3 個
- [ ] 外部權威來源連結 1-2 個
- [ ] 行動裝置友善段落長度（≤150 字）

## 輸出格式

根據要求輸出：
- **完整草稿**：可直接使用的全文
- **大綱 + 重點**：讓使用者自行填充
- **多版本 A/B**：標題 / CTA 的不同版本
