# Marketing — 行銷策略規劃

## 觸發條件

- 規劃產品上市（GTM）或行銷活動
- 設計 A/B 測試或成長實驗
- 分析競品定位與市場機會
- 制定內容行銷策略
- 評估行銷 KPI 與 ROI

## 四步驟流程

### Step 1：情境釐清（Situation）

需要確認：
- **目標**：這次行銷想達成什麼？（品牌知名度 / 獲客 / 留存 / 營收）
- **受眾**：目標 ICP（Ideal Customer Profile）的輪廓
- **限制**：預算、時間、通路偏好、禁忌

### Step 2：策略設計（Strategy）

**定位框架**：
```
我們幫助 [目標受眾]
解決 [核心問題]
不像 [競品]
我們的做法是 [差異化優勢]
```

**通路選擇矩陣**：
| 通路 | 適合目標 | 成本 | 速度 |
|------|---------|------|------|
| SEO / 內容行銷 | 長期品牌 / 有機流量 | 低 | 慢 |
| Paid Search (SEM) | 獲客 / 高意向 | 高 | 快 |
| Social Media | 品牌認知 / 社群 | 中 | 中 |
| Email 行銷 | 留存 / 再行銷 | 低 | 快 |
| KOL / Influencer | 信任建立 / 觸及 | 中-高 | 中 |
| PR / Media | 品牌聲量 | 低 | 慢 |

### Step 3：執行規劃（Execution）

**GTM Checklist**：
- [ ] 確認核心訊息（1 句話的 value proposition）
- [ ] 準備分眾素材（不同受眾版本）
- [ ] 設定追蹤機制（UTM / Pixel / Conversion）
- [ ] 制定內容日曆（發佈時間 / 頻率）
- [ ] 準備 Landing Page / CTA
- [ ] 設定 A/B 測試假設

**A/B 測試設計模板**：
```
假設：我們相信 [變更] 會讓 [指標] 提升 [X%]
因為 [理由]

測試設計：
- 控制組（A）：[現有版本]
- 測試組（B）：[改動版本]
- 樣本大小：[計算所需人數]
- 測試期間：[最少 2 週，避免星期效應]
- 成功指標：[主要 KPI + 護欄指標]
```

### Step 4：量測與優化（Measurement）

**KPI 框架（依目標）**：
| 目標 | 主要 KPI | 次要 KPI |
|------|---------|---------|
| 品牌認知 | 曝光數、觸及率 | 品牌搜尋量 |
| 獲客 | CPA、轉換率 | LTV:CAC 比 |
| 留存 | Churn Rate | NPS、DAU/MAU |
| 營收 | MRR 成長、ARPU | 升級率 |

## 輸出格式

```markdown
## 行銷策略摘要

**目標**：[一句話]
**受眾**：[ICP 描述]
**核心訊息**：[value proposition]

### 建議通路組合
1. [通路 1] — 理由：[X]，預期效果：[Y]
2. [通路 2] — ...

### A/B 測試提案
[假設 + 設計]

### 90 天里程碑
- 第 1 個月：[聚焦項目]
- 第 2 個月：[優化項目]
- 第 3 個月：[擴大項目]

### KPI 追蹤表
[指標 / 基準 / 目標]
```

## Gotcha

- **沒有策略就開始執行**：先花 30 分鐘做情境釐清，能省下數週的錯誤嘗試
- **KPI 太多**：一個活動選 1-2 個 North Star Metric，其餘是護欄
- **A/B 測試結束太早**：統計顯著性需要足夠樣本（建議用 Evan Miller 計算機）
- **忽略護欄指標**：A 指標提升但 B 指標下跌，不算成功
