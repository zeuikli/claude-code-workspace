# PM — 專案管理

## 觸發條件

- 啟動新專案或 Sprint
- 任務優先排序困難
- 需要撰寫狀態報告 / 向利害關係人更新進度
- Sprint 結束做回顧
- 評估專案風險

## 核心工作流

### Sprint 規劃（雙週版）

```markdown
## Sprint [X] 規劃

**目標**：[一句話，Sprint 結束後要達成什麼]
**期間**：[日期 ~ 日期]

### 納入項目
| Story | 點數 | 負責人 | 驗收條件 |
|-------|------|-------|---------|
| [任務 1] | X | [人] | [可驗證條件] |

**Total**：X 點 / Sprint 容量 Y 點

### 排除（Backlog）
- [任務 A] — 理由：[X]
```

### 優先排序

**MoSCoW 框架**：
```
Must Have   — 沒有就不能上線（核心功能、合規）
Should Have — 重要但可以延後（影響 UX）
Could Have  — 有的話更好（nice to have）
Won't Have  — 本次不做（記錄在 backlog）
```

**RICE 評分**（數字化比較）：
```
Reach (觸及人數/週): X
Impact (影響度 0.25/0.5/1/2/3): X
Confidence (信心度 %): X%
Effort (工程人週): X

RICE = (Reach × Impact × Confidence) / Effort
```

### 狀態報告模板

```markdown
## 專案狀態 — [日期]

**整體狀態**：🟢 正常 / 🟡 注意 / 🔴 風險

### 本週完成
- [具體交付物 1]
- [具體交付物 2]

### 下週計畫
- [計畫項目 1]
- [計畫項目 2]

### 風險 & 障礙
| 風險 | 可能性 | 影響 | 緩解措施 |
|------|-------|------|---------|
| [風險 1] | 高/中/低 | 高/中/低 | [行動] |

### 需要決策
- [決策 1] — 截止日期：[X]
```

### 利害關係人更新（電梯版）

```markdown
【專案 X 快報】

✅ 本週完成：[1-2 句話]
🚀 下週重點：[1-2 句話]
⚠️ 需要你的輸入：[具體問題]

如有疑問請在 [日期] 前回覆。
```

### Sprint 回顧（Start/Stop/Continue）

```markdown
## Sprint [X] 回顧

**Start**（應該開始做）
- [改進 1]

**Stop**（應該停止做）
- [問題 1]

**Continue**（應該繼續做）
- [優點 1]

**Action Items**
| 行動 | 負責人 | 截止 |
|------|-------|------|
| [改進行動] | [人] | [日期] |
```

### 風險矩陣

```
高影響 │ 🟡 監控  │ 🔴 立即處理
       │         │
低影響 │ ✅ 接受  │ 🟡 計畫中
       ├─────────┼──────────
         低可能性   高可能性
```

## Gotcha

- **只更新，不決策**：狀態報告要清楚標示「需要利害關係人決策」的事項
- **容量規劃**：Sprint 容量留 20% buffer（病假、臨時插件）
- **Definition of Done**：每個 Story 要有明確驗收條件，避免「差不多完成」
- **回顧要有 Action Items**：沒有 owner + deadline 的回顧決議等於沒有
